Understanding the Concepts

🎯
Bias
HOW FAR OFF ON AVERAGE
Think of it as: How wrong is your model on average? Bias measures the gap between your model's average prediction and the true answer.
Analogy: Imagine throwing darts at a target. High bias = your darts consistently land in the wrong area (far from the bullseye), even if they're grouped together. The aim itself is off.
FORMULA
Bias = E[ŷ] − ytrue
// Average prediction minus the true value

Bias² = (E[ŷ] − ytrue)²
// We square it so direction doesn't matter
High bias → model is too simple → it underfits → misses the real pattern
🎲
Variance
HOW MUCH PREDICTIONS JUMP AROUND
Think of it as: How much does your model's prediction change when trained on different data? Variance measures how sensitive the model is to the specific training set it saw.
Analogy: Back to darts — High variance = your darts are scattered everywhere. Each throw lands in a very different spot. The model is unstable.
FORMULA
Var = E[(ŷ − E[ŷ])²]
// How spread out are predictions
// around their own average?
High variance → model is too complex → it overfits → memorizes noise
📐
The Total Error Equation
WHY YOU CAN'T MINIMIZE BOTH AT ONCE
Total Error = Bias² + Variance + σ²
E[(y − ŷ)²] = (E[ŷ] − ytrue)² + E[(ŷ − E[ŷ])²] + σ²
Bias²
Error from wrong assumptions.
Decreases with complexity.
Variance
Error from sensitivity to data.
Increases with complexity.
σ² (Irreducible)
Noise in the data itself.
Can never be eliminated.
The tradeoff: As you make your model more complex, bias drops but variance rises. The sweet spot is where their sum is minimized — that's the U-shaped curve you see in the chart above.

SECTION 06 · BAGGING

Bagging & Random Forests

Bootstrap AGGregating — train many trees on random subsets, then average their votes

The Bootstrap Trick
For each tree, randomly sample ~63% of the training data with replacement. Each tree sees a different version of reality — so each learns different patterns.
Random Feature Selection
At each split, consider only a random subset of features (typically √n features). This further decorrelates the trees and prevents any one strong feature from dominating.
Why Averaging Helps
Each tree has high variance (it's sensitive to its training subset). But averaging many high-variance models dramatically reduces the overall variance — without increasing bias.

Boosting — Learning from Mistakes

Each weak learner focuses on the errors of all previous learners. Click to step through.

SEQUENTIAL LEARNER CHAIN:
Boosting Intuition
Start with a weak model. Find what it got wrong. Train the next model specifically on those hard examples. Repeat — each model fills the gaps of the previous ones.
SAMPLE WEIGHTS (harder samples get higher weight):
AdaBoost vs Gradient Boosting
AdaBoost
Re-weights misclassified samples. Learners are decision stumps (depth-1 trees).
Gradient Boosting
Fits each learner to the residual errors (gradients) of the previous ensemble. Powers XGBoost, LightGBM, CatBoost.

Regularization Techniques

Four ways to prevent overfitting — click each tab to explore interactively

L1 Regularization (Lasso)
Adds the sum of absolute values of weights to the loss: Loss = Loss + λ·Σ|w|
Effect: Drives some weights to exactly zero — automatic feature selection. Only the most important features survive.
λ (regularization strength):
No penaltyStrong penalty
Weight magnitudes (grey = zeroed out):
WHEN TO USE
When you suspect many input features are irrelevant. L1 will zero them out automatically, giving you a sparse, interpretable model.
ANALOGY
Imagine paying rent for every feature you use. L1 charges full price — so you only keep features worth paying for.
IN PRACTICE
Start with λ = 0.001. Increase until you see the desired sparsity. Monitor which features get zeroed.
L2 Regularization (Ridge / Weight Decay)
Adds the sum of squared values of weights: Loss = Loss + λ·Σw²
Effect: Shrinks all weights proportionally toward zero but rarely to exactly zero. Keeps all features but small.
λ (regularization strength):
Weight magnitudes (all shrink proportionally):
L1 vs L2
L1 penalty|w|
L2 penaltyw²
Squaring punishes large weights heavily but lets small weights survive — none reach exactly zero.
ANALOGY
Progressive tax — the bigger your weight, the more you pay. Everyone stays in the game, but no one gets too powerful.
IN DEEP LEARNING
L2 is often called weight decay in deep learning frameworks. Adam + weight decay = AdamW, the default for training LLMs.
Dropout
Randomly deactivates neurons during training. Each forward pass uses a different subset of the network.
Dropout rate (fraction of neurons dropped):
0% (no dropout)80% (heavy dropout)
Active neuron Dropped out
INTUITION
Training a sports team where random players sit out each practice. Every player learns every role — no one becomes indispensable.
TRAINING vs INFERENCE
During training: neurons randomly dropped. During inference: all neurons active, outputs scaled proportionally by 1 - dropout_rate.
TYPICAL VALUES
Dense layers: 0.3–0.5
Conv layers: 0.1–0.2
Too high → underfitting
Early Stopping
Monitor validation loss. Stop training when it stops improving. Save the weights from the best checkpoint.
PATIENCE
How many epochs to wait before accepting no improvement. Typical: patience = 10 epochs.

Too low → stops prematurely. Too high → wastes compute.
WHY IT WORKS
The model is most generalized just before it starts memorizing. Early stopping freezes it at that peak moment.
COST
Essentially free. No extra computation, no new hyperparameters beyond patience. Try this first!
